# Инструкция: Добавление колонки shares_outstanding

## Шаг 1: Выполнить SQL миграцию в Supabase

### 1.1. Откройте Supabase Dashboard
1. Перейдите на https://supabase.com/dashboard
2. Войдите в свой аккаунт
3. Выберите проект FindGreatStocks (production)

### 1.2. Откройте SQL Editor
1. В левом меню найдите раздел **"SQL Editor"** (иконка с символом `</>` или текст "SQL Editor")
2. Нажмите на него

### 1.3. Создайте новый запрос
1. Нажмите кнопку **"New query"** (или "+ New query") вверху
2. В открывшемся редакторе вы увидите пустое поле для SQL

### 1.4. Скопируйте и вставьте SQL скрипт
1. Откройте файл `migrations/add-shares-outstanding.sql` в вашем проекте
2. Выделите весь текст (Ctrl+A) и скопируйте (Ctrl+C)
3. Вставьте в SQL Editor в Supabase (Ctrl+V)

### 1.5. Запустите скрипт
1. Нажмите кнопку **"Run"** (или "Execute", или F5) внизу справа
2. Дождитесь завершения выполнения (должно появиться сообщение "Success" или "Query executed successfully")
3. Если появились ошибки — проверьте, что все таблицы существуют

### 1.6. Проверьте результат
1. В левом меню найдите **"Table Editor"**
2. Откройте любую таблицу (например, `companies`)
3. Прокрутите вниз — должна появиться новая колонка: `shares_outstanding`

---

## Что делает эта миграция?

1. **Добавляет колонку `shares_outstanding`** во все таблицы компаний:
   - `companies`
   - `sp500_companies`
   - `nasdaq100_companies`
   - `dow_jones_companies`
   - `ftse100_companies`

2. **Создает индексы** для оптимизации запросов

3. **Позволяет системе**:
   - Сохранять количество акций в обращении для каждой компании
   - Рассчитывать market cap = price * shares_outstanding
   - Обновлять market cap автоматически при изменении цены

---

## После выполнения миграции

Система автоматически начнет:
- Получать `shares_outstanding` из Yahoo Finance или FMP API
- Сохранять его в базу данных
- Использовать для расчета `market_cap` при обновлении цен

Следующее обновление цен (через scheduler или вручную) заполнит эту колонку для всех компаний.
