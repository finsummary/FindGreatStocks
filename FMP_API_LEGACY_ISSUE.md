# Проблема с FMP API Legacy Endpoints

## Текущая ситуация

После обновления с `/api/v3/` на `/api/v4/` все еще получаем ошибку:

```
"Legacy Endpoint : Due to Legacy endpoints being no longer supported - 
This endpoint is only available for legacy users who have valid subscriptions 
prior August 31, 2025."
```

Это означает, что **v4 тоже является legacy эндпоинтом**.

## Решение

Нужно проверить актуальную документацию FMP API:
https://site.financialmodelingprep.com/developer/docs

### Возможные варианты:

1. **FMP использует другой формат URL** - возможно не `/api/v3/` или `/api/v4/`, а что-то вроде:
   - `/api/quote/` (без версии)
   - `/v1/quote/` (другой формат)
   - `https://api.financialmodelingprep.com/v1/quote/` (другой домен)

2. **Изменен формат аутентификации** - возможно нужно:
   - Передавать API ключ в заголовках вместо query параметра
   - Использовать Bearer token
   - Другой формат авторизации

3. **Нужна подписка на новый план** - возможно старые ключи не работают с новыми эндпоинтами

## Что делать:

1. **Проверьте документацию FMP:**
   - Зайдите на https://site.financialmodelingprep.com/developer/docs
   - Найдите актуальные эндпоинты для quote и profile
   - Проверьте формат URL и аутентификации

2. **Проверьте ваш FMP аккаунт:**
   - Убедитесь, что у вас активная подписка
   - Проверьте, поддерживает ли ваш план новые эндпоинты
   - Возможно, нужно обновить план подписки

3. **Протестируйте новый формат:**
   - Попробуйте сделать запрос напрямую с новым форматом
   - Проверьте, какой формат работает

## Временное решение

Если новые эндпоинты недоступны, можно:
- Использовать альтернативный источник данных для цен
- Обновить подписку FMP на план, который поддерживает новые эндпоинты
- Связаться с поддержкой FMP для получения информации о миграции
